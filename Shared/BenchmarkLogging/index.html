Minimum Shader Startup Test

  • The startup test is an even smaller app for testing if WebGL works on any platform.
  • An internet connection is necessary during the benchmark.
  • Shader performance may vary, some GPU configurations may execute the stress tests faster than the ordinary benchmark.
  • Run the benchmark for at least 1 minute as performance often is slower at first.
  • Some plugins may under certain conditions affect WebGL performance. Consider disabling Firebug, Greasemonkey and similar plugins if accurate benchmarking performance is needed.
  • Any development computer must be able to run these benchmarks without major visual artifacts
  • We consider interactive framerates on the main benchmark to be sufficient for development computers.
  • Interactive frame rates on the shader stress test (especially High resolution version) is not necessary for our purpouses.
  • If any development computer runs the stress test benchmarks other development computers should be largely unaffected.
